# 会议秘书与图像识别功能 - 技术方案

## 🎯 功能1：会议秘书

### 功能定位
**让TOKI成为你的会议助理，自动记录、整理、跟进**

### 核心功能

#### 1. 会议记录
- **实时语音转文字**
  - 识别多个发言人
  - 区分发言者（主持人、参会者）
  - 自动标记时间戳

- **关键内容提取**
  - 决策事项
  - 行动项（Action Items）
  - 待办事项
  - 重要观点

#### 2. 会议纪要生成
- **自动生成模板**
  ```
  会议纪要
  
  📅 会议时间：2026-04-09 14:00-16:00
  📍 会议地点：3楼会议室
  👥 参会人员：张三、李四、王五...
  
  一、会议议题
  1. Q2销售目标讨论
  2. 新产品发布计划
  
  二、决策事项
  ✅ Q2销售目标：提升20%
  ✅ 新产品发布时间：6月15日
  
  三、行动项（Action Items）
  | 负责人 | 任务 | 截止时间 |
  |--------|------|----------|
  | 张三 | 制定销售计划 | 4月15日 |
  | 李四 | 准备产品资料 | 4月20日 |
  | 王五 | 联系供应商 | 4月12日 |
  
  四、下次会议
  时间：4月16日 14:00
  议题：销售计划评审
  
  ---
  会议记录：TOKI AI
  记录时间：2026-04-09 16:00
  ```

#### 3. 任务跟进
- **自动创建提醒**
  - 根据Action Items自动创建提醒
  - 分配给对应负责人
  - 截止日期提醒

- **执行计划生成**
  ```
  Q2销售提升20%执行计划
  
  第一阶段（4月）：市场调研
  - 目标：完成市场分析报告
  - 负责人：张三
  - 关键节点：4月10日
  
  第二阶段（5月）：策略制定
  - 目标：确定销售策略
  - 负责人：李四
  - 关键节点：5月5日
  
  第三阶段（6月）：执行推进
  - 目标：执行销售计划
  - 全员参与
  - 检查点：每周五
  ```

### 技术实现方案

#### 整体架构
```
会议音频流
    ↓
实时语音识别（ASR）
    ↓
说话人分离（Diarization）
    ↓
内容理解与提取（NLU）
    ↓
纪要生成（Generation）
    ↓
任务分发（Task Distribution）
```

#### 技术栈

**1. 实时语音识别**
- **方案A：科大讯飞实时语音转写**
  - 实时性：< 300ms
  - 准确率：98%+
  - 支持说话人分离
  - 价格：¥0.008/秒

- **方案B：阿里云实时语音识别**
  - 实时性：< 500ms
  - 准确率：96%+
  - 价格：¥0.006/秒

- **方案C：自建模型（Whisper + Diarization）**
  - 实时性：1-2秒延迟
  - 准确率：95%+
  - 需要GPU服务器

**推荐：科大讯飞实时语音转写**

**2. 说话人分离（Speaker Diarization）**
```
技术方案：
- 使用科大讯飞说话人分离API
- 或使用pyannote.audio（开源）
- 识别准确率：90%+
```

**3. 内容理解与提取**
```python
prompt = """
你是会议秘书，分析会议内容并提取关键信息。

会议记录：
{meeting_transcript}

请提取以下信息：
1. 决策事项
2. 行动项（负责人、任务、截止时间）
3. 待办事项
4. 下次会议安排

返回JSON格式：
{
  "decisions": [...],
  "action_items": [
    {
      "owner": "张三",
      "task": "制定销售计划",
      "deadline": "2026-04-15"
    }
  ],
  "todos": [...],
  "next_meeting": {...}
}
"""
```

## 🎯 功能2：图像识别与视频通话

### 功能定位
**精准识别图像内容，支持实时视频通话分析**

### 核心功能

#### 1. 图像识别
- **通用物体识别**
  - 识别图片中的物体
  - 分类标签
  - 位置标注

- **OCR文字识别**
  - 图片转文字
  - 表格识别
  - 手写文字识别

- **场景理解**
  - 图片内容描述
  - 细节分析
  - 异常检测

#### 2. 视频通话功能
- **实时视频分析**
  - 实时识别视频中的人物
  - 实时OCR
  - 实时场景理解

- **视频会议增强**
  - 实时字幕
  - 说话人识别
  - 关键信息标注

### 技术实现方案

#### 整体架构
```
图像/视频输入
    ↓
图像预处理
    ↓
多模态模型识别
    ↓
内容理解与分析
    ↓
结果输出
```

#### 技术栈

**1. 多模态模型选择**

**方案A：GPT-4 Vision（推荐）**
- 识别准确率：⭐⭐⭐⭐⭐
- 细节理解：优秀
- 文字识别：优秀
- 价格：$0.01/次
- **问题：不支持视频实时分析**

**方案B：Google Gemini Pro Vision**
- 识别准确率：⭐⭐⭐⭐⭐
- 细节理解：优秀
- 支持视频分析
- 价格：$0.0025/次
- **推荐用于视频**

**方案C：Gemma 4（Tony提到的模型）**
```
Gemma 4特点：
- Google开源模型
- 多模态能力
- 可本地部署
- 需要GPU资源

但注意：
❗ Gemma 4还未正式发布
❗ 当前最新版本是Gemma 2
❗ Gemma 2的多模态能力有限
```

**实际推荐方案**：
- **图像识别**：GPT-4 Vision 或 Qwen-VL-Plus
- **视频分析**：Google Gemini Pro Vision
- **成本优化**：Qwen-VL-Plus（¥0.008/次）

**2. OCR识别**
```
方案A：百度OCR
- 准确率：99%+
- 价格：¥0.002/次
- 免费：每日500次

方案B：腾讯云OCR
- 准确率：98%+
- 价格：¥0.0015/次

方案C：PaddleOCR（开源）
- 准确率：95%+
- 免费
- 可本地部署
```

**推荐：百度OCR（高准确率） + PaddleOCR（免费备选）**

### 技术难点

**难点1：100%颗粒度识别**
```
现实情况：
❌ 没有100%准确率的模型
✅ GPT-4 Vision可达95%+准确率
✅ 结合OCR可达98%+文字识别率

建议：
- 多模型结合
- 人工复核关键信息
- 持续优化模型
```

**难点2：视频实时分析**
```
技术挑战：
- 视频流处理需要高算力
- 实时性要求高
- 成本较高

解决方案：
- 抽帧分析（每秒1-2帧）
- 使用Gemini Pro Vision
- 云端GPU加速
```

**难点3：Gemma 4模型**
```
澄清：
❗ Gemma 4尚未发布
❗ 当前最新是Gemma 2
❗ Gemma 2多模态能力有限

建议：
- 等待Gemma 4正式发布
- 当前使用GPT-4V或Gemini Pro
- 关注Google官方动态
```
